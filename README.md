# BigData Snowflake Lab

Лабораторная работа: нормализация исходных CSV-данных в аналитическую модель типа снежинка.
## Структура репозитория

```
├── docker-compose.yml       # Конфигурация контейнера и точка входа для SQL-скриптов
├── исходные данные/         # 10 CSV-файлов (по 1000 строк в каждом)
└── sql/
    ├── 01_ddl.sql           # Создание схем, таблиц измерений и фактов
    ├── 02_dml_load_staging  # Импорт CSV-файлов в staging-слой
    ├── 03_dml_transform.sql # ETL-процесс: трансформация данных в витрину dwh
    └── 04_checks.sql        # Контрольные и аналитические запросы
```

## Развёртывание и запуск

Для автоматического создания базы данных, загрузки данных и построения витрины выполните:

```bash
docker compose up -d
```

### Параметры подключения

| Параметр   | Значение      |
|------------|---------------|
| Host       | `localhost`   |
| Port       | `5433`        |
| Database   | `bdsnowflake` |
| User       | `postgres`    |
| Password   | `postgres`    |

### Управление состоянием
- При **первом старте** контейнер последовательно выполнит скрипты из папки `sql/`: создаст схему, загрузит 10 000 строк в `staging.mock_data` и запустит процесс трансформации в `dwh`.
- Для **полного сброса** (удаление volume и повторная загрузка с нуля) используйте:
  ```bash
  docker compose down -v
  docker compose up -d
  ```

## Архитектура данных

Реализована модель «снежинка» с чётким разделением на слой фактов и нормализованные измерения.

**Центральная таблица фактов:**
- `dwh.fact_sales` — транзакции продаж

**Измерения (1-й уровень нормализации):**
- `dwh.dim_customer` (покупатели)
- `dwh.dim_seller` (продавцы)
- `dwh.dim_product` (товары)
- `dwh.dim_store` (магазины)
- `dwh.dim_supplier` (поставщики)
- `dwh.dim_date` (календарь)

**Справочники (2-й уровень, ветвления снежинки):**
Страны, типы питомцев, категории питомцев, категории товаров, бренды и материалы товаров.

## Валидация результата

После завершения работы контейнера рекомендуется выполнить контрольные запросы для проверки целостности данных:

```sql
SELECT COUNT(*) FROM staging.mock_data;
SELECT COUNT(*) FROM dwh.fact_sales;
```

Ожидаемый результат для обоих запросов: `10000`. Расхождение в количествах строк указывает на ошибки в логике JOIN или фильтрации на этапе трансформации.
